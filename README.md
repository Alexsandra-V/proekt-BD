# proekt-BD
Проект: база данных "Онлайн школа английского языка"

О проекте

Этот проект представляет собой реляционную базу данных, разработанную для моделирования работы онлайн-школы английского языка. Система отслеживает:

· Учеников и их профили (личные данные, уровень владения языком, прогресс обучения).
· Преподавателей и их загрузку (ведение курсов и проведение уроков).
· Учебные курсы и их структуру (название, количество уроков, цена, уровень сложности).
· Уровни владения языком (от начального до продвинутого).
· Расписание занятий (уроки) с указанием даты, времени и преподавателя.
· Связь преподавателей с курсами (какие преподаватели ведут какие курсы).
· Запись учеников на курсы (какие ученики на каких курсах обучаются).

База данных может служить основой для реальной системы управления онлайн-школой, использоваться для автоматизации записи на курсы, формирования расписания, отслеживания прогресса учеников и расчета заработной платы преподавателей. Также проект может быть использован в учебных целях для изучения проектирования реляционных моделей, нормализации данных и работы со связями "многие-ко-многим".

Сущности (таблицы)

· ycheniki (Ученики): Основная информация об учениках: ID, имя, телефон, данные банковской карты (для оплаты), уровень владения языком, текущий курс, закрепленный преподаватель (куратор) и ссылка на профиль ученика.
· yroven (Уровни): Уровни владения английским языком (например, Beginner, Elementary, Pre-Intermediate и т.д.).
· kyrsi (Курсы): Учебные курсы: ID, название, количество уроков в курсе, цена, уровень сложности (связь с таблицей уровней).
· ychitel (Преподаватели): Информация о преподавателях: ID, имя, телефон.
· yroki (Уроки): Расписание занятий: дата, время начала, продолжительность (в минутах), курс, преподаватель.
· profil_ychenika (Профиль ученика): Учебный прогресс ученика: количество пройденных уроков и количество завершенных курсов.
· prepodov_v_kyrse (Преподаватели в курсе): Связь «многие-ко-многим» между преподавателями и курсами (один преподаватель может вести несколько курсов, один курс могут вести несколько преподавателей).
· ychenik_v_kyrse (Ученик в курсе): Связь «многие-ко-многим» между профилями учеников и курсами (один ученик может обучаться на нескольких курсах, на одном курсе может быть несколько учеников).

Связи между сущностями

Схема связей построена на принципах реляционной целостности и включает следующие зависимости:

1. yroven ← kyrsi
   Каждый курс соответствует определенному уровню владения языком.
   (kyrsi.id_yrovnya → yroven.id_yrovnya)
2. yroven ← ycheniki
   Каждому ученику назначен текущий уровень владения языком.
   (ycheniki.id_yrovnya → yroven.id_yrovnya)
3. kyrsi ← ycheniki
   Ученик может быть зачислен на определенный курс (как на основной).
   (ycheniki.id_kyrsa → kyrsi.id_kyrsa)
4. ychitel ← ycheniki
   За учеником закреплен курирующий преподаватель.
   (ycheniki.id_ychitel → ychitel.id_ychitel)
5. kyrsi ← yroki
   Каждый урок принадлежит определенному курсу.
   (yroki.id_kyrsa → kyrsi.id_kyrsa)
6. ychitel ← yroki
   Каждый урок проводит определенный преподаватель.
   (yroki.id_ychitel → ychitel.id_ychitel)
7. profil_ychenika ← ycheniki
   Каждый ученик имеет единственный профиль для отслеживания прогресса.
   (ycheniki.id_profilya → profil_ychenika.id_profilya)
8. kyrsi ↔ ychitel (многие-ко-многим)
   Связь между курсами и преподавателями реализована через связующую таблицу prepodov_v_kyrse.
   (prepodov_v_kyrse.id_kyrsa → kyrsi.id_kyrsa; prepodov_v_kyrse.id_ychitel → ychitel.id_ychitel)
9. profil_ychenika ↔ kyrsi (многие-ко-многим)
   Связь между учениками (через их профили) и курсами, на которые они записаны, реализована через связующую таблицу ychenik_v_kyrse.
   (ychenik_v_kyrse.id_profilya → profil_ychenika.id_profilya; ychenik_v_kyrse.id_kyrsa → kyrsi.id_kyrsa)
